## Unified UI Approach (Cribbed From the Book)

All three documents share a minimal UI toolkit:

1. **Full-screen toggle** — a small button (⛶ or similar) top-right of each embedded tutorial section. Clicking expands that tutorial to fill the viewport (hides surrounding white paper content), click again to return. Uses `position: fixed; inset: 0; z-index: 999; overflow-y: auto; background: inherit;` — pure CSS + ~5 lines JS.

2. **Expand/Collapse All** — for accordion sections (borrowed from book's expand-toggle pattern).

3. **Hover panels** — CSS tooltips with `data-tip` for inline terms. Same pattern as book's `.hover-term` but simplified to pure CSS (no JS positioning needed at our scale).

4. **Pause/Play on animations** — consistent button style across all documents.

5. **Progressive disclosure** — `<details><summary>` as base, enhanced with JS for smooth transitions.

6. **Dark/light respect** — `prefers-color-scheme` media queries. Magnetosphere tutorial is dark-themed by design; inference tutorial and white paper are light. When embedded, each section carries its own background.

7. **Self-download** — one button, downloads the current page as a single HTML file.

This gives a unified feel across all three pieces without imposing heavy framework machinery.

3. THE SPACE — Tokens become vectors. Distance = similarity. Position is blended in. Meaning has geometry.
4. THE CONVERSATION — Attention: each token asks "who matters to me?" Context is computed, not stored.
5. THE CHOICE — Probability distribution over vocabulary. Temperature. Weighted coin flip. One token chosen. Repeat.
